## 1. 安装与初次设置

### 1.1 安装糖盾

从GitHub下载最新APK文件，在手机上安装。

🔗 https://github.com/jiang213658168/TangDun

安装时如果提示"未知来源"，请前往 **设置 → 安全 → 允许安装未知应用**，勾选你的浏览器或文件管理器。

### 1.2 激活码

首次打开糖盾，会要求输入**激活码**。

> 激活码是您购买后获得的一串以 `TD2.` 开头的字符串。
> 如果您还没有激活码，请联系开发者获取。

**激活步骤：**
1. 打开糖盾App
2. 阅读并同意用户协议
3. 在弹出的对话框中输入激活码
4. 点击"激活"
5. 激活成功后，会显示您的用户类型和有效期

**激活码类型：**
| 类型 | 说明 |
|------|------|
| 管理员 | 永久有效，无任何限制 |
| 普通用户 | 有时效性，每日各功能有使用次数 |

### 1.3 授予必要权限

安装后首次使用，会引导您授予必要权限：

| 权限 | 用途 | 必须？ |
|------|------|--------|
| **通知访问权限** | 读取CGM动态血糖仪的通知栏数据 | ✅ 必须 |
| 后台运行权限 | 保持App在后台持续接收血糖 | ✅ 必须 |
| 相机权限 | 拍照识别食物 | 可选 |
| 电话权限 | 严重低血糖时自动拨打紧急联系人 | 可选 |

**如何开启通知访问权限：**
1. 进入手机 **设置 → 通知与状态栏 → 通知使用权限**（不同手机路径略有差异）
2. 找到 **糖盾**
3. 打开开关

> ⚠️ **这个权限很重要！** 不开的话，糖盾无法自动获取血糖数据，您只能手动录入。

## 2. 连接血糖仪数据

糖盾支持三种方式获取血糖数据。您只需使用其中一种即可。

### 方式一：通知栏自动读取（推荐，最简单）

> 适用于：欧态健康(Aidex)、Dexcom、雅培Libre、微泰等**40+品牌**

**操作步骤：**
1. 确保您手机上的CGM App正在运行（如欧态健康App）
2. 确保CGM App的**通知栏显示血糖数值**（通常默认开启）
3. 打开糖盾 → 授予通知访问权限
4. 糖盾会自动读取通知栏中的血糖值

> 无需任何额外设置。只要CGM App在通知栏显示了血糖，糖盾就能自动接收。

**工作流程：**
```
CGM传感器 → CGM App → 通知栏显示血糖值
              ↓
         糖盾自动读取 → 保存 → 预测 → 预警
```

### 方式二：xDrip+广播接收

> 适用于：已安装xDrip+的用户

**操作步骤：**
1. 在xDrip+中：**设置 → Inter-app settings → ✅ Broadcast locally → ✅ Compatible Broadcast**
2. 糖盾会自动接收xDrip+的广播数据

### 方式三：手动输入

> 适用于：没有CGM设备，使用指尖血糖仪的用户

**操作步骤：**
1. 在糖盾首页，点击右上角的 **➕** 按钮
2. 选择血糖来源：指尖血
3. 输入血糖值
4. 选择测量场景（空腹/餐前/餐后/睡前/其他）
5. 点击"保存"

## 4. 指尖校准：让数据更准

> CGM传感器读数有时会与指尖血有偏差。校准可以让糖盾自动修正这个偏差。

### 如何校准

1. 用指尖血糖仪测量一次血糖
2. 在糖盾首页，找到**指尖校准**卡片
3. 点击 **"录入指尖值校准"**
4. 输入指尖血糖仪显示的数值
5. 点击 **"确认"**

糖盾会自动：
- 比较指尖值与当前CGM值
- 计算补偿偏移量
- 之后所有CGM读数都会自动加上这个偏移

### 校准状态

| 显示 | 含义 |
|------|------|
| 未校准 | 还没有录过指尖值 |
| 已校准1次 | 偏移= +1.2 | 已校准1次，当前偏移量 |
| 已校准10次 | 偏移= +0.3 | 多次校准后更稳定 |

**建议：** 每天校准1-2次，或当感觉CGM数值不准时校准。多次校准后，偏移量会自动平滑。

## 5. 预测页面：提前看未来2小时

点击底部导航栏的 **"预测"** 进入。

### 页面构成

```
┌──────────────────────────┐
│     血糖预测          🔄 │
├──────────────────────────┤
│  风险等级：正常          │
│  当前：6.8 mmol/L        │
├──────────────────────────┤
│  预测引擎                │
│  TCN 40% | DallaMan(7室) 60% │
│  置信度：60% | 记录288条  │
│  IOB 1.2U | 碳水 85g     │
├──────────────────────────┤
│  血糖预测曲线            │
│  ┌────────────────────┐  │
│  │  (历史+预测)       │  │
│  └────────────────────┘  │
│  当前 6.8 | 30min 8.2    │
│  60min 9.5 | 120min 7.8  │
├──────────────────────────┤
│  预计峰值 9.5 @ 60分钟后 │
└──────────────────────────┘
```

### 预测引擎信息

| 指标 | 含义 |
|------|------|
| **TCN权重** | 深度学习模型的权重。数据越多越高 |
| **DallaMan权重** | 生理模型的权重。数据少时更高 |
| **置信度** | 综合评分。>60%为可靠 |
| **IOB** | 活性胰岛素。体内还剩多少胰岛素在起作用 |
| **今日碳水** | 今天记录的碳水化合物总量 |
| **ISF/CR** | 估算的胰岛素敏感因子和碳水系数 |

### 预测曲线的含义

- **青色实线** = 过去24小时的实测血糖
- **橙色虚线** = AI预测的未来2小时血糖
- **绿色区域** = 你的目标范围
- **红色圆点** = 当前血糖

### 阅读预测结果

**正常：** 预测30分钟后在目标范围内  
**高血糖风险：** 预测30分钟后将超出目标上限 → 注意饮食+考虑补针  
**低血糖风险：** 预测30分钟后将低于目标下限 → 建议提前补充碳水  

> 💡 **预测不是100%准确的。** MAE为0.552 mmol/L意味着平均误差约0.5 mmol/L。结合自己的感受和指尖校准结果综合判断。

## 6. 记录饮食/胰岛素/运动

点击底部导航栏的 **"记录"** 进入。

这里有四个入口：

### 📝 饮食记录

记录每餐的食物和碳水含量。

**方式一：手动输入**
1. 点击"饮食"
2. 选择餐型（早餐/午餐/晚餐/加餐）
3. 输入食物名称、碳水克数、热量
4. 点击保存

**方式二：拍照识别**（需要网络 + 百度AI配置）
1. 点击相机图标
2. 拍摄食物照片
3. 等待AI识别（约3-5秒）
4. 自动填充食物名称、GI值、碳水、热量
5. 确认并保存

### 💉 胰岛素记录

记录每次胰岛素注射。

1. 点击"胰岛素"
2. 选择胰岛素类型（速效/短效/长效/预混）
3. 输入剂量（单位）
4. 选择注射部位
5. 可选：关联饮食记录
6. 点击保存

> ⚠️ 记录胰岛素后，糖盾会自动刷新预测。预测会考虑活性胰岛素(IOB)。

### 🏃 运动记录

1. 点击"运动"
2. 选择运动类型（走路/跑步/骑车/游泳等）
3. 输入时长
4. 可选：输入心率、步数
5. 点击保存

> 记录运动后，糖盾会在预测中考虑运动降糖效应。

### ❤️ 健康记录

记录体重、血压、睡眠、酮体等健康数据。

## 7. AI助手：智能问答与拍照识食

### 进入方式

- **首页**：点击顶栏的 💬 图标
- **记录页**：点击"AI助手"卡片

### AI对话功能

可以直接向AI提问，例如：

> "我晚餐吃了200克米饭，餐后2小时血糖12.5，正常吗？该怎么办？"  
> "运动前血糖多少比较安全？"  
> "低血糖时应该吃什么？吃多少？"  
> "我的糖化血红蛋白7.2%，需要调整方案吗？"

AI助手基于大语言模型，知识涵盖1型/2型糖尿病管理、血糖监测、胰岛素使用、碳水计数、运动管理、并发症预防等。

> ⚠️ **AI建议仅供参考**，不能替代医生。涉及药物剂量调整请务必咨询专业医生。

### 配置AI服务

1. 进入 **设置 → AI对话配置**
2. 糖盾默认已配置AI服务（兼容OpenAI的API），可直接使用
3. 如需使用自己的API密钥，可在设置中修改

## 8. 报告页面：周报月报一键生成

点击底部导航栏的 **"报告"** 进入。

### 报告内容

- **日报**：当天血糖统计、TIR、GRI、饮食/胰岛素/运动汇总
- **周报**：7天趋势、平均TIR、血糖变异度
- **月报**：HbA1c估算、月度趋势图、改进建议

### 分享报告

在 **设置 → 数据分享** 中，点击"分享今日报告"，可以生成文本报告并发送给微信好友、医生、或保存为文件。

## 9. 设置页面

点击底部导航栏的 **"我的"** 进入。

### 重要设置项

| 设置项 | 说明 | 建议值 |
|--------|------|--------|
| **血糖目标范围** | 低值和高值 | 3.9-10.0 mmol/L |
| **严重低/高阈值** | 触发紧急告警的值 | 3.0 / 13.9 mmol/L |
| **身高体重** | 用于个性化预测参数 | 如实填写 |
| **紧急联系人** | 严重低血糖时自动拨打 | 家人/伴侣电话 |
| **AI对话配置** | AI助手的API设置 | 默认已配置 |
| **通知设置** | 告警声音/振动开关 | 全部打开 |

### 自学习状态

在设置页顶部可以查看糖盾的**自学习状态**：

- **初始阶段**：收集数据中（<3天）
- **冷启动阶段**：开始学习你的模式（3-14天）
- **稳定阶段**：个性化优化完成（>14天）

## 10. 常见问题

### Q: 糖盾为什么没有自动收到血糖数据？

**检查清单：**
1. ☐ 通知访问权限是否已授予？（设置→通知→通知使用权限→糖盾）
2. ☐ CGM App是否正在运行？（欧态健康/Dexcom等）
3. ☐ CGM App的通知栏是否显示血糖数值？
4. ☐ 手机是否开启了省电模式？（省电模式可能杀死后台服务）
5. ☐ 尝试下拉首页手动刷新

### Q: 糖盾显示的血糖值和CGM App不一样？

这是正常的。原因：
1. CGM App显示的可能是原始值，糖盾应用了**指尖校准偏移**
2. 如果偏差超过0.5 mmol/L，建议做一次**指尖校准**来修正

### Q: 预测曲线显示"下降"但我实际血糖在上升？

可能原因：
1. 你刚录入了饮食但还没切换回预测页 → 下拉刷新
2. 活性胰岛素较多 → 糖盾认为胰岛素效应大于碳水效应
3. 你的饮食记录中碳水克数填少了
4. 建议：确保饮食记录准确，尤其是碳水克数

### Q: App被杀掉了怎么办？

糖盾会自动重启。你会在通知栏看到"糖盾监测中"的持久通知。如果通知消失了，重新打开糖盾即可恢复。

### Q: 如何备份数据？

1. 进入 **设置 → 数据备份**
2. 点击"备份数据"
3. 备份文件保存在 `内部存储/Android/data/com.tangdun.app/files/backups/`

### Q: AI助手无法使用？

1. 检查网络连接
2. 进入 **设置 → AI对话配置**，确认API密钥已配置
3. 免费API可能有每日使用次数限制（详见您的激活码说明）

### Q: 系统提示"电池优化"或"后台限制"？

部分手机（尤其是华为/小米/OPPO）会自动限制后台应用。

**解决方法：**
1. 进入 **设置 → 应用 → 糖盾 → 电池 → 无限制**
2. 进入 **设置 → 应用 → 糖盾 → 自启动 → 允许**
3. 在手机管家/安全中心中，将糖盾加入白名单
